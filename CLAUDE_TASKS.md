# Claude 修改任務清單



## 6個HTML檔案說明

|檔案名|用途|重點|
|-|-|-|
|Dashboard|個人化導航儀表板|整合各項工具入口、時鐘與常用連結導航|
|Secretary|生活小秘書與日程管理|記錄待辦事項、提醒與日常行政瑣事追蹤|
|Inventory|物品清單與庫存管理|分類記錄家裡物品數量、效期與補充提醒|
|Wardrobe|電子衣櫃與穿搭管理|數位化管理服飾、季節分類與穿搭記錄|
|Pocket List|口袋名單地圖收藏|儲存想去的餐廳、景點或值得收藏的資訊|
|M.Y Wiki|靈感收藏與知識庫|系統化整理設計靈感、技術筆記與知識積累|



## 📝 7個任務清單



### 任務 1: 嘟嘟一家首頁 (Dashboard) 優化需求

\*\*修改檔案\*\* ：https://raw.githubusercontent.com/yui0151/home/main/index.html
\*\*目前\*\* ：各區域使用的是較明顯的「垃圾桶 🗑️」圖示，且需確認跳轉功能狀態。
\*\*期望修改\*\* ：

1. **圖示更換** ：將所有區域（待辦事項、心願清單、重要紀念日）原本的「垃圾桶 🗑️」圖示，統一更換為更不明顯、簡約的「x」圖示。
2. **風格維持** ：確保「x」圖示符合整體的日系手繪簡約風格。
3. **導航確認** ：確保此頁面作為核心，能正確跳轉至其他子分頁。
原因/說明：維持整體日系手繪的簡約質感，並確保核心入口導航正常。

### 任務 2: 家務秘書 (Secretary) 優化需求

\*\*修改檔案\*\* ：https://raw.githubusercontent.com/yui0151/secretary/main/index.html
\*\*目前\*\* ：列表使用垃圾桶圖示，繳費與定期任務缺乏每兩個月的設定選項，且頂部存在重複的導航圖示。
\*\*期望修改\*\* ：

1. **圖示更換** ：比照首頁，將列表中的「垃圾桶」圖示更換為「x」。
2. **新增功能** ：在繳費/定期任務設定中，新增「每兩個月」的選項。
3. **邏輯優化** ：新增選項後，需比照「每月 x 日」的邏輯，自動計算並生成下一個截止日期。
4. **刪除重複導航** ：刪除頂部的「房子」按鈕（統一由底部導航列進行切換）。
原因/說明：統一全站視覺圖示，擴充定期任務功能的實用性與自動化，並精簡多餘的導航元件。

### 任務 3: 庫存管家 (Inventory) 優化需求

\*\*修改檔案\*\* ：https://raw.githubusercontent.com/yui0151/inventory/main/index.html

\*\*期望修改\*\* ：

1. **頁面順序交換** ：將「庫存」與「待買清單」兩個頁籤位置對調。
2. **預設顯示** ：設定進入庫存頁面時，初始畫面直接顯示「待買清單」。
3. **刪除重複導航** ：刪除頂部的「房子」按鈕（統一由底部導航列進行切換）。
原因/說明：提升使用者的操作直覺與效率，快速查看需要補充的物品，並保持介面清爽。

4. **刪除重複導航** ：刪除頂部的「房子」按鈕（統一由底部導航列進行切換）。

### 任務 6: 口袋地圖 (Pocket List) 優化需求

\*\*修改檔案\*\* ：https://raw.githubusercontent.com/yamap610/pocket-list/main/index.html
\*\*目前\*\* ：搜尋功能無法搜尋帶有標籤（#）的內容，且存在 UI 截斷與跑版問題。
\*\*期望修改\*\* ：

1. **搜尋功能增強** ：修復搜尋框無法搜尋帶有「#（標籤）」內容的問題。
2. **UI 修正** ：修復頂部搜尋欄輸入範例被截斷的問題，並修正詳細資訊彈窗或底部的版面跑版問題。
3. **導航確認** ：確保下方導航列與首頁一致且功能正常。
原因/說明：強化標籤搜尋的精準度，並修復介面顯示異常。

### 任務 7: 全局共用優化 (Global Fix)

\*\*修改檔案\*\* ：所有 HTML 檔案
\*\*目前\*\* ：各子頁面的底部導航欄容易遮擋內容，且需維持全站一致的視覺風格與單一導航邏輯。
\*\*期望修改\*\* ：

1. **版面防遮擋 (CSS Padding/Margin)** ：針對所有頁面進行 CSS 檢查，在底部導航欄 (Tab Bar) 上方預留足夠的 padding/margin，確保滑動到底部時，內容不會被導航欄遮擋。
2. **統一導航邏輯** ：確保所有子分頁皆已移除頂部多餘的「房子」按鈕，完全依賴底部導覽列進行頁面切換與返回首頁。
3. **視覺風格** ：全站維持柔和的奶茶色調、日系簡約風格與手繪感元件。
原因/說明：確保整個系統的導航一致性，徹底解決底部內容被遮擋的痛點，並維持高度統一的視覺美感。
