# Claude 修改任務清單



## 6個HTML檔案說明
| 檔案名 | 用途 | 重點 |
|--------|------|------|
| Dashboard | 個人化導航儀表板 | 整合各項工具入口、時鐘與常用連結導航 |
| Secretary | 生活小秘書與日程管理 | 記錄待辦事項、提醒與日常行政瑣事追蹤 |
| Inventory | 物品清單與庫存管理 | 分類記錄家裡物品數量、效期與補充提醒 |
| Wardrobe | 電子衣櫃與穿搭管理 | 數位化管理服飾、季節分類與穿搭記錄 |
| Pocket List | 口袋名單地圖收藏 | 儲存想去的餐廳、景點或值得收藏的資訊 |
| M.Y Wiki | 靈感收藏與知識庫 | 系統化整理設計靈感、技術筆記與知識積累 |



## 📝 6個任務清單

### 任務 1: 修復「嘟嘟一家」首頁資料同步與 UI 精進
**修改檔案**：`https://raw.githubusercontent.com/yui0151/home/main/index.html` [cite: 6]
**目前**：
* 頁面開啟時資料偶爾消失或未從 Firebase 完整抓取 [cite: 4]。
* 新增項目後重新整理會導致資料重置 [cite: 4]。
* 子分頁跳轉會開啟新分頁，且缺乏自動存檔機制 [cite: 4]。
* 缺乏整合各系統資訊（衣櫃、家務秘書、口袋名單）的智慧提醒功能 [cite: 4]。
**期望修改**：
1. **修復資料消失**：檢查 `init` 邏輯，確保頁面開啟時從 Firebase 完整抓取所有舊資料（待辦、備忘、心願、紀念日） [cite: 4]。
2. **資料持久化**：修正存入邏輯，確保新增項目後「重新整理」資料不會重置 [cite: 4]。
3. **編輯與拖曳**：家庭待辦、心願清單需支援「原地編輯 (Inline Edit)」、「原地刪除 (Inline Delete)」與 SortableJS 拖曳排序 [cite: 4]。
4. **UI 精進**：編輯、刪除、拖曳符號需極小、淡色（低不透明度），維持日系質感 [cite: 4]。
5. **單一 App 體驗**：移除所有跳轉連結的 `target="_blank"`，改為同分頁跳轉，且跳轉前需強制執行 `saveDash()` 自動存檔 [cite: 4]。
6. **專屬管家日報 (Morning Briefing)**：結合問候語與各系統資料，例如：「早安！今天變涼了，記得幫小嘟拿件『薄外套』（連動衣櫃）；家務秘書提醒您今天有『信用卡費』要繳喔！」 [cite: 4]
7. **天氣與出遊智能提案**：當抓取到今日是「晴天 ☀️」或週末時，自動從「口袋地圖」隨機挑選標籤為「親子放電」或「戶外景點」且「尚未造訪」的項目 [cite: 4]。
8. **衣櫃斷捨離提醒**：當「小嘟衣櫃」裡被標記為「穿不下」的衣服累積超過 5 件，首頁待辦自動置頂提醒 [cite: 4]。
**原因/說明**：提升首頁作為核心入口的資訊整合度與操作流暢感，並強化與其他子系統的連動性。

### 任務 2: 「家務秘書」新增「每兩個月定期繳納」自動化功能
**修改檔案**：`https://raw.githubusercontent.com/yui0151/secretary/main/index.html` [cite: 7]
**目前**：僅能設定每月固定日期，且需手動更改日期以生成下個月項目 [cite: 4]。
**期望修改**：
1. **新增「定期繳納」設定**：除了每個月外，支援設定「每兩月」固定幾號繳費 [cite: 4]。
2. **自動化邏輯**：項目設定後，當本月完成勾選，系統應自動帶入下個月（或下一個週期）同日期的待繳項目，不需手動修改 [cite: 4]。
3. **日期溢位處理**：若該月份沒有該日期（如 31 號），則自動跳至該月最後一天 [cite: 4]。
**原因/說明**：減少重複性行政瑣事，確保週期性帳單不被遺漏。

### 任務 3: M.Y 小百科 (M.Y Wiki) 優化需求
**修改檔案**：`https://raw.githubusercontent.com/yamap610/mywiki/main/index.html` [cite: 9]
**目前**：分類選項受限且缺乏自動化連結預覽功能 [cite: 4]。
**期望修改**：
1. **更新類別**：預設選單新增「📁 未分類」 [cite: 4]。
2. **自動預覽**：貼上連結後自動抓取 Open Graph (og:image) 圖片；若有手動上傳則優先顯示上傳圖 [cite: 4]。
3. **分類管理**：設定頁支援修改類別名稱與 Emoji，並同步更新已存項目 [cite: 4]。
**原因/說明**：優化設計靈感與技術筆記的整理效率。

### 任務 5: 口袋地圖 (Pocket List) 優化需求
**修改檔案**：`https://raw.githubusercontent.com/yamap610/pocket-list/main/index.html` [cite: 9]
**目前**：備註顯示格式跑掉、詳細頁面滑動卡頓、搜尋功能受限且缺乏快速營業時間輸入 [cite: 4]。
**期望修改**：
1. **內容格式**：備註欄位需保留使用者的換行與空格 [cite: 4]。
2. **UI 滑動優化**：修復新增/編輯頁面的標題被擋住問題；詳細頁面視窗需滑動絲滑，並修復右上角「X」無法按下的問題 [cite: 4]。
3. **營業時間強化**：詳細頁面需顯示營業時間；新增頁面應支援 Enter 鍵輸入 [cite: 4]。
4. **標籤搜尋**：搜尋功能需支援搜尋 Tag 內容（例如搜「中山」可出現有 #中山 的店家） [cite: 4]。
5. **表格化營業時間**：開發能迅速新增營業時間的功能，並使其呈現如 Google Map 般的表格樣式 [cite: 4]。
**原因/說明**：解決多項 UX 痛點，使其更接近專業地圖收藏工具的使用感。

### 任務 6: 全域體驗與 PWA 升級基礎
**修改檔案**：上述所有 6 個 HTML 檔案（Dashboard, Secretary, Inventory, Wardrobe, Pocket List, M.Y Wiki） [cite: 1, 6, 8]
**目前**：各頁面導航不統一，依賴跳轉新分頁且未有存檔檢查機制 [cite: 4]。
**期望修改**：
1. **全域底部導覽列 (Bottom Navigation)**：在所有頁面最下方加入固定式圖示導覽列（🏠, 🐧, 🐰, 👗, 🗺️, 📖） [cite: 4]。
2. **導航優化**：移除所有 `target="_blank"`，改為同視窗跳轉，並在跳轉前檢查輸入中的資料是否已存檔 [cite: 4]。
3. **UI 清理**：移除各子分頁原有的「←首頁」按鈕，功能統一由底部導覽列承接 [cite: 4]。
**原因/說明**：建立單一應用程式（Single App）的封閉體驗，確保資料安全且切換流暢。
