Claude 修改任務清單 (嘟嘟一家 2.0 升級版)
============================================================

[專案概述]
- 目標：優化為手機優先、可打包為 PWA 的原生 App 體驗。
- 技術棧：HTML/JS/CSS (Firebase: stockapp-a1ccc)。
- 核心視覺：日系奶茶色、奶油色系、大圓角、極簡質感（拖曳符號需低不透明度）。

------------------------------------------------------------
[核心優化：全域與連動架構 (Task 0)]
------------------------------------------------------------
1. 全域底部導覽列 (Bottom Navigation)：
   - 於所有 HTML 頁面底部新增固定導航欄。
   - 選項：[首頁、家務、庫存、衣櫃、百科、地圖]。
   - 移除所有 target="_blank"，改為同分頁跳轉，實現單一 App 體驗。

2. 自動存檔防護：
   - 跳轉前攔截事件，強制觸發 Firebase 存檔邏輯。
   - 加入 beforeunload 或 pagehide 事件，防護瀏覽器退回動作。

3. 專屬管家日報 (Morning Briefing)：
   - 首頁問候語需動態結合資料庫。
   - 範例：[氣溫下降，記得幫小嘟拿「薄外套」(連動衣櫃)]。
   - 範例：[今天有「信用卡費」要繳(連動家務)]。

------------------------------------------------------------
[5 大分頁任務清單]
------------------------------------------------------------

## 任務 1: 首頁 (index.html) 資料同步與智能連動
- 數據修復：修正 init 邏輯，確保重整頁面不丟失 Firebase 舊資料。
  (涵蓋：待辦、備忘、心願、紀念日)
- 心願一鍵轉移：新增「移至待買」按鈕。
  (點擊後自動從首頁移除，並寫入「庫存管家」的待買資料庫)
- 智能出遊提案：若天氣為「晴天」或週末，隨機抓取「口袋地圖」。
  (條件：標籤為「親子放電/戶外」且「尚未造訪」之景點)
- 衣櫃斷捨離提醒：當衣櫃中「穿不下」衣物 >= 5 件，首頁待辦自動置頂。
- 編輯體驗：支援 SortableJS 拖曳排序與原地編輯 (Inline Edit)。
- 備援機制：天氣定位失敗預設為「桃園市」。

## 任務 2: M.Y 小百科 (encyclopedia.html) 功能補全
- Bug 修復：正確綁定 Firebase ID，確保點擊卡片可進入詳細頁、編輯或刪除。
- 自動預覽圖：優先顯示手動上傳圖，無圖則抓取網址 og:image。
- 分類管理：預設新增「手作靈感」、「飲食健康」。
- 類別編輯：設定頁支援類別編輯（名稱/Emoji）並連動更新舊標籤。
- 排序切換：新增按鈕支援依「時間」、「標題 A-Z」、「過期日」排序。

## 任務 3: 家務秘書 (secretary.html) 每月自動化
- 定期繳納設定：新增欄位 isRecurring: true, dayOfMonth: [1-31]。
- 自動生成邏輯：當月任務勾選完成後，自動生成下個月同日期的項目。
- 大小月處理：若設定 31 號但下個月無該日期，自動跳至該月最後一天。

## 任務 4: 跨系統導航與返回機制
- 返回按鈕：各子頁面 Header 新增極簡「返回首頁」按鈕，觸發前強制存檔。
- UI 優化：下拉選單加大以利手機操作，所有切換動作需平滑無阻。
